Workout Program UX Philosophy
## Workout Program UX Philosophy

Design the workout program experience around **simplicity, continuity, and progress tracking**, not around complex fitness science.

The system must support two roles:

- Trainer
- Member

Both should interact with the same workout program from different perspectives.

---

# Trainer Experience (Program Creation)

The trainer's goal is to create a workout program as quickly as possible.

The workflow should feel similar to writing a workout plan on paper, but with the advantages of a digital system.

The trainer should not be forced to understand templates, training cycles, periodization, mesocycles, or advanced fitness terminology.

The experience must remain practical and intuitive.

### Trainer Workflow

1. Open "Create Workout Program"
2. Select Program Type:
    - General Program
    - Private Program
3. If Private Program is selected:
    - Select a member
4. Enter Program Name
5. Create Day 1
6. Enter Day Title

Examples:

- Push Day
- Pull Day
- Legs Day
- Chest + Triceps
- Full Body

7. Add exercises to the day
8. For each exercise define:
    - Sets
    - Reps
9. Save the day
10. Add additional workout days
11. Save the entire program

---

### Important UX Principles

The trainer thinks in workout days.

The trainer does not think in:

- Weeks
- Calendar dates
- Monday, Tuesday, Wednesday
- Training cycles

Therefore the system should be organized as:

Day 1

Day 2

Day 3

Day 4

and so on.

The program should be built as a sequence of workout days.

---

### General Program UX

General Programs act as reusable workout plans.

A trainer can create:

- Beginner Program
- Weight Loss Program
- Push Pull Legs Program
- Advanced Program

These programs are available for assignment later.

---

### Private Program UX

Private Programs are assigned directly to a specific member.

The workflow remains identical.

The only difference is selecting a member before saving.

The trainer should not feel that they are using a different system.

---

# Member Experience (Program Execution)

The member's goal is not to read the program.

The member's goal is to perform the next workout.

The UX should always answer one question:

"What should I do right now?"

---

### Member Workflow

1. Open the application
2. Immediately see the current workout

Example:

Today:

Day 2 — Pull Day

4 Exercises Remaining

Start Workout

---

3. Start the workout session
4. Complete exercises one by one
5. Record:
    - Weight Used
    - Actual Reps
6. Mark sets as completed
7. Continue until all exercises are completed
8. Finish the workout session

---

### Workout Progress UX

The member should always know:

- Which workout day they are currently on
- Which workout days have been completed
- Which workout day comes next

A visual progress timeline should be displayed.

Example:

✓ Day 1 — Push Day

Current → Day 2 — Pull Day

Upcoming → Day 3 — Legs Day

---

### Workout History UX

For every exercise, the member should see previous performance.

Example:

Bench Press

Last Workout:

60 kg × 10

65 kg × 8

65 kg × 8

This allows the member to attempt progression naturally.

---

### Workout Completion UX

When all exercises are completed:

The session should be marked as completed.

Display:

- Workout Duration
- Completed Exercises
- Total Volume
- Performance Summary

The user should feel a sense of achievement.

---

# Core UX Principle

The trainer manages the plan.

The member executes the plan.

The trainer focuses on:

- Creating workout days
- Choosing exercises
- Assigning programs

The member focuses on:

- Completing today's workout
- Recording weights
- Tracking progress

The system should always guide both users toward the next action instead of overwhelming them with information.

The entire workout experience should feel simple, fast, motivating, and professional.
